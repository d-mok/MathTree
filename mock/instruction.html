Content Editing

  1. The content editor can be used as MSWord.
  2. The symbol '*' signify a variable. e.g. '*x' will be substituted by the value of x from code.
  3. LaTex is used in equation objects. See KaTeX for a list of supported syntax.
  4. Answer options in MC must be listed as bullet points. e.g.
    • I only
    • I and II only
    • I and III only
    • I, II and III only
    The first option must be the correct answer.

Variable: Populate Code

  1. All variables must be defined here.
  2. All alphabets (a-z,A-Z) are pre-declared as variables. Once defined, their values will be substituted in the content.
  3. Variables can be number or string. Decimals will be rounded to 5 sig. fig. before substitution.
  4. Each line should end with a semicolon ';'.

Variable: Validate Code

  1. All variables validation should be done here.
  2. The code here must be a single boolean expression, no matter how complex.
  3. The code here can access the alphabet variables (a-z,A-Z) defined earlier.
  4. One line should contain only one condition for clarity.
  5. If no validation is needed, just left it blank.
  6. Some common logical operators:
    • and: x && y
    • or: x || y
    • not: !x
    • equal: x === y
    • unequal: x !== y

Processing Code

  1. This code is for manipulating the content as html strings.
  2. This code can access the alphabet variables (a-z,A-Z) defined earlier.
  3. This code can access the question html string as question.
  4. This code can access the solution html string as solution.
  5. Preprocess code are executed before substituting alphabet varaibles.
  6. Postprocess code are executed after substituting alphabet varaibles.

Code Execution Flow

  1. Run the population code to define to variables.
  2. Run the validation code.
    If failed, populate again and again until validation passed. The execution will stop after 1000 failed trials.
  3. Run the preprocess code.
  4. Substitute the variables, and do some smart adjustment:
    • Round all decimal variables to 5 significant figures.
    • Correct minus signs, e.g. x--y become x+y.
    • Correct unit coefficient, e.g. x+1y become x+y.
  5. Run the postprocess code.
  6. Shuffle the answer options.
    If duplicated options are found, the whole execution will restart from step 1. The execution will stop after 1000 failed trials.
